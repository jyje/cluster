# open-webui SQLite → PostgreSQL Migration

## Overview

Migrate open-webui's internal SQLite database (`webui.db`) to a CloudNativePG (CNPG)
PostgreSQL cluster to enable multi-replica HA deployment.

| Item | Value |
|------|-------|
| Date | 2026-05-03 |
| Source | `PVC: open-webui` → `/data/webui.db` (SQLite, ~9 MB) |
| Target | CNPG cluster `open-webui-pg` (3 instances, `openwebui` DB) |
| Namespace | `ollama-system` |
| Credential secret | `open-webui-pg-app` (auto-generated by CNPG, key: `uri`) |

---

## Background

open-webui stores all state in SQLite by default:
- Chat history, user accounts, model configs, API keys
- File stored at `/app/backend/data/webui.db` on the `open-webui` PVC (10 Gi RWX)

**Problems with SQLite in Kubernetes:**
- Not safe under concurrent writes (multi-replica blocked)
- SQLite over NFS (subdir-usb StorageClass) causes intermittent `database is locked` errors
- OOMKill restarts leave WAL files in inconsistent state

## Pre-Migration Changes (committed to git)

### 1. Deploy CNPG cluster (`open-webui-pg`)

Added to `clusters/r4spi/apps/open-webui.yaml` extraResources:

```yaml
- apiVersion: postgresql.cnpg.io/v1
  kind: Cluster
  metadata:
    name: open-webui-pg
  spec:
    instances: 3
    storage:
      size: 2Gi
    bootstrap:
      initdb:
        database: openwebui
        owner: openwebui
    failoverDelay: 30
    postgresql:
      parameters:
        wal_sender_timeout: 60s
        wal_receiver_timeout: 60s
        wal_keep_size: 2048MB
```

CNPG auto-generates `open-webui-pg-app` secret with `uri` key.

### 2. Scale down open-webui + wire DATABASE_URL

```yaml
replicaCount: 0
extraEnvVars:
  - name: DATABASE_URL
    valueFrom:
      secretKeyRef:
        name: open-webui-pg-app
        key: uri
```

## Migration Steps

### Step 1 — Scale down open-webui

```bash
# Triggered via git commit: replicaCount: 0
# ArgoCD applied → open-webui-0 pod terminated
```

### Step 2 — Verify CNPG cluster health

```bash
kubectl get cluster -n ollama-system open-webui-pg
# Expected: 3/3 · Cluster in healthy state
```

### Step 3 — Apply migration Job

```bash
kubectl apply -f /tmp/owui-migration-job.yaml
```

The Job consists of:

**InitContainer `create-schema`** — Run Alembic to create PostgreSQL schema:
```bash
cd /app/backend/open_webui && python -m alembic upgrade head
```

**Container `migrate-data`** — Python script (`psycopg2`) copies data:
1. Reads all tables from SQLite via `sqlite3`
2. Gets boolean column info from PostgreSQL schema (`information_schema`)
3. Converts SQLite `0/1` integers → Python `bool` for boolean columns
4. Disables FK constraints (`SET session_replication_role = replica`)
5. `TRUNCATE … CASCADE` + bulk insert via `psycopg2.extras.execute_batch`
6. Resets PostgreSQL sequences after copy

### Step 4 — Verify migration results

```bash
kubectl logs -n ollama-system -l job-name=owui-sqlite-to-pg -c migrate-data
```

### Step 5 — Scale up open-webui with PostgreSQL

```bash
# git commit: replicaCount: 1
# ArgoCD applies → open-webui starts with DATABASE_URL pointing to PostgreSQL
```

## Troubleshooting Log

### Attempt 1 — Alembic path wrong
- **Error**: `No 'script_location' key found`
- **Fix**: Changed working dir from `/app/backend` to `/app/backend/open_webui`

### Attempt 2 — Missing env var
- **Error**: `ValueError: Required environment variable not found`
- **Fix**: Added `WEBUI_SECRET_KEY` from `openwebui-creds` secret to init container

### Attempt 3 — asyncpg not available
- **Error**: `ModuleNotFoundError: No module named 'asyncpg'`
- **Fix**: Rewrote migration script using `psycopg2` (sync) instead of `asyncpg`

### Attempt 4 — session_replication_role requires superuser
- **Error**: `InsufficientPrivilege: permission denied to set parameter "session_replication_role"`
- **Fix**: Replaced `SET session_replication_role` with `ALTER TABLE ... DISABLE TRIGGER ALL`

### Attempt 5 — DISABLE TRIGGER ALL also requires superuser for constraint triggers
- **Error**: FK violations on `chat_file` and `group_member` (alphabetical order broke FK deps)
- **Fix**: Implemented Kahn's topological sort using `information_schema` FK dependency graph.
  Parents are inserted before children, eliminating FK violations without needing superuser.

## Rollback Plan

If migration fails or open-webui behaves unexpectedly after switching to PostgreSQL:

1. Remove `DATABASE_URL` env var from `open-webui.yaml`
2. Set `replicaCount: 1`
3. Commit + push → ArgoCD reverts to SQLite-backed deployment
4. The original `webui.db` on the PVC is untouched (mounted read-only during migration)
